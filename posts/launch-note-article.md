## 1. なぜ「AIを、使わせろ」なのか

「AI を使わせろ」は、AI を要求しているのではない。**AI を握る側と握られる側の構造を反転させろ**、と言っている。

OpenAI、Anthropic、Google ── 巨大資本が API という蛇口を握り、いくらで使わせるかを決めている。これ自体を問題視しているわけではない。問題は、その蛇口を**叩ける個人**と**叩けない個人**の格差が、AI 自体の進化と並行して**指数関数的に開いている**ことだ。

資本を持つ者ほど AI を持っている。AI を持つ者がさらに資本を増やす。このループに入れない大多数は、5 年後には**取り戻せない断絶**の向こう側に置き去りにされる。

「AI を学ぶべき」「AI に投資せよ」では足りない。**個人が個人として AI を使い倒し、巨大資本の経済圏に対抗する圏域を作る** ── これが「AIを、使わせろ」という叫びの実体である。

## 2. 弁証法による設計

私は LP を設計するときに、必ず**弁証法**(正・反・合)で構造を組む。  
今回もそうした。

### 正 ── 思想 LP として作る

主張: メッセージの拡散性が最優先。**単一 HTML、CDN 配信、フォーク歓迎、改変自由**。
強み: 即時拡散、誰でもクローン可能、CDN 落ちても残る。
弱み: 決済・認証・会員 DB・解析が組めない。「真の経済圏」を名乗るには弱い。

### 反 ── 事業 LP として作る

主張: 本番運用に耐える完全スタック。**Next.js + 認証 + 決済 + 解析**を一体化。
強み: 観測性・継続更新・収益化、すべてが揃う。
弱み: 拡散性消失、特定プラットフォームに人質、改変フォーク不可。「中央集権の否定」を掲げる任侠思想と矛盾する。

### 合 ── 三層分離

両者を 1 つにしようとするから矛盾する。**責務で層を分ければ、両方を犠牲にせず統合できる**。

```
L1  思想層    Static HTML · GitHub Pages · CC0    → 拡散性
L2  事業層    Vite · React · TS · Tailwind        → 運用堅牢性
L3  プロトコル層  Solidity · Base/Arbitrum         → 不可逆性(任侠の自動執行)
```

L1 で叫び、L2 で運用し、L3 で仁義を貫く。  
**単一スタックで矛盾を抱えるな。層を分けて、責務で統合せよ**。

## 3. 七箇条 ── 経済圏の不可侵宣言

LP の核は **七箇条**である。

1. AIは握るものではない。**使い倒すもの**である。
2. **弱者から先に**、AIを渡せ。
3. 義のないAIは、ただの暴力である。
4. 経済圏の**天頂**に、個人を立たせる。
5. 情報の格差は、**奪い返す**。
6. 仁義は、**コードで貫く**。
7. そして、**叫び続けろ**。AIを、使わせろ。

特に重要なのは第六と第七である。

第六「**仁義は、コードで貫く**」── これが L3 プロトコル層の存在理由になる。同志を喰わない、利を独占しない、声を上げ続ける。これらをスマートコントラクトで自動執行する。違反すれば自動で破門。口約束ではなく、**プロトコルで強制する**。

第七「**叫び続けろ**」── これがいま、この note 記事を書いている動機である。沈黙する者から AI は奪われる。書き続け、叫び続けることが、第七箇条の実装である。

---

## 4. 三つの仁義(Doctrines)

七箇条が**宣言**だとすれば、三つの仁義は**作法**である。

- **義** ── 同志を、喰わない(同じ経済圏の仲間を AI で収奪しない)
- **仁** ── 利を、独り占めしない(AI で得た利益の一部を経済圏に還元する)
- **勇** ── 声を、上げ続ける(沈黙せず、外向きに発信し続ける)

これに違反した者は、たとえ親分であっても破門となる。経済圏の作法は、**シンプルで明確**でなければならない。複雑な規約は守られないし、抜け道で食い物にされる。

---

## 5. なぜ CC0 なのか

第五箇条「**情報の格差は、奪い返す**」を、**ライセンスで実装した**結果が CC0 である。

CC0(Creative Commons Zero)は、著作権を可能な限り放棄するパブリックドメイン宣言だ。誰でも、どんな目的でも、複製・改変・再配布・商用利用が自由にできる。

この LP の HTML、コピー、構造、デザイン、すべて誰のものでもない。  
別の経済圏を立てたいなら、`s/TENCHO_AI_ECONOMY/お前の経済圏/g` で書き換えて使えばいい。それが第五箇条の正しい実装である。

「自分の作ったものを守りたい」という発想は、自分が握る側に回りたい欲望の裏返しだ。**握らないことが、構造を反転させる第一歩である**。

## 6. フォークの作法

フォークは歓迎する。だが、いくつかの作法がある。

1. **思想を借りるなら、思想を継げ** ── 七箇条の精神に違反する使い方はするな。
2. **謝意は不要、引用は任意** ── 「TENCHO の弟分」ぶる必要はない。お前の経済圏として立てろ。
3. **違う思想なら、違う旗を立てろ** ── 中身まで真似する必要はない。骨組みだけ流用して、お前の血を流し込め。

フォーク先のリンクを送ってくれれば、L1 の README に相互リンクとして並べる。同じ思想なら同志として、違う思想なら良きライバルとして。
